# Frontend Migration Guide - Product Module Refactoring

## 1. Overview

The `smartLive-marketing` module has been refactored and merged into `smartLive-product`. All 

Voucher related concepts are now unified under 

Product

. This guide details the API changes, data structure updates, and integration points for the frontend.



## 2. API Changes

### 2.1 Base URL

- **Old**: `/marketing/voucher` (or similar)

- New

  :

   

  ```
  /product/
  ```

  - Example: `GET /product/list` (Query product list)
  - Example: `POST /product` (Add product)
  - Example: `GET /product/{id}` (Get details)

### 2.2 Search API

- Endpoint

  :

   

  ```
  POST /search/vouchers
  ```

   

  (This endpoint name remains for compatibility but returns

   

  ProductDoc

   

  structure).

- **Index**: Backend now queries `product_index`.

## 3. Data Structure Changes

### 3.1 Product Entity (Form Submission)

When creating or editing a product, use the following payload structure. Note the `category` field.

```
json






{

  "id": 123,

  "shopId": 1,

  "name": "Product Name",

  "subTitle": "Subtitle",

  "category": 1,          // 1: Voucher (代金券), 2: Set Meal (团购套餐)

  "activityType": 0,      // 0: Normal Sales, 1: Seckill Activity

  "price": 100.00,        // Selling Price (Yuan)

  "originalPrice": 200.00,// Original Price (Yuan)

  "stock": 100,

  "rulesJson": "...",     // JSON string for rules

  "beginTime": "2026-02-18 10:00:00", // For Seckill

  "endTime": "2026-02-19 10:00:00"    // For Seckill

}
```

### 3.2 Search Result (ProductDoc)

The search results from `/search/vouchers` have the following fields. **Note the unit difference for prices.**

| Field         | Type    | Description                                             |
| :------------ | :------ | :------------------------------------------------------ |
| id            | Long    | Product ID                                              |
| `title`       | String  | Product Name                                            |
| `subTitle`    | String  | Subtitle                                                |
| `shopName`    | String  | Shop Name                                               |
| `payValue`    | String  | **Price in Cents** (e.g., "10000" = 100.00 Yuan)        |
| `actualValue` | Long    | **Original Price in Cents** (e.g., 20000 = 200.00 Yuan) |
| `type`        | Integer | 0: Normal, 1: Seckill                                   |
| `rules`       | String  | Usage rules (mapped from `rulesJson`)                   |

> **Important**: The search result `payValue` is a String representing cents. You must divide by 100 to display Yuan.

## 4. Enum Constants

A new global enum `ProductEnum` has been added. Frontend should use these values for `category`.

- Voucher

   

  = 1

- `Set Meal` = 2

## 5. Action Items

1. **Update API Paths**: Replace all `/marketing/voucher` prefixes with `/product/product`.
2. **Update Field Binding**: Ensure forms bind to `category` instead of previous type fields if any.
3. **Handle Price Units**: In search result text, ensure `payValue` (cents) is converted to Yuan for display.
4. **Test**: Verify search, creation, and purchasing flows.
